# E224 - table layout conflicts with commit_row_mode shared

## Cause

The resolved theme has `commit_label_layout: table` while the author
explicitly set `commit_row_mode: shared`. Table layout lays one commit
per row (its rows *are* the commit-axis rows), so it requires
`unique` row packing. Leaving `commit_row_mode` unset is fine — table
mode then forces `unique` on its own — but an explicit `shared`
directly contradicts what table mode needs, so the contradiction is
flagged rather than silently overridden.

The trigger is an *explicit* `commit_row_mode: shared`. A `shared`
value inherited from a theme default (rather than set by the author)
does not trip it.

## Example

```
{"op": "theme", "commit_label_layout": "table", "commit_row_mode": "shared"}
```

## How to fix

Pick one of:

- Drop the `commit_row_mode` field — table mode forces `unique`
  automatically.
- Set `commit_row_mode: unique` explicitly (matching what table mode
  needs).
- Use `commit_label_layout: inline` if you actually want shared rows.
