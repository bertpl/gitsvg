# E306 - `import` is not available for in-memory input

## Cause

The op stream was provided as an in-memory string — for example via
the `gitsvg.render_text()` Python API — rather than read from a file.
String input has no filesystem location, so an `import` op has no
directory to resolve against and no containment root; imports are
rejected outright for it, regardless of the path.

## Example

```
{"op": "import", "path": "./base.gitsvg.jsonl"}
```

(when passed to `render_text` as a string rather than read from a
file)

## How to fix

Inline the imported file's ops directly into the string, or switch to
file-based input (`gitsvg render <file>`), where imports resolve
relative to the importing file.
