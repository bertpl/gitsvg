# E201 - Reference to undefined commit

## Cause

A field that takes a commit id (`branch.from_commit`, `commit.replaces`,
`merge.as`, `highlight.commit`, `remove.commits`) refers to a commit that
doesn't exist in current state. Either the id was never declared, or it has
been removed by an earlier `remove` op or a `replaces:` squash.

When the same name exists as a branch, the message hints that you
probably meant the branch field instead (e.g. `from_branch` rather
than `from_commit`). When the id looks like a typo of a declared
commit, the error carries that commit id as its `suggestion` (rendered
as `did you mean '<id>'?`, and available structurally in `--json`).

## Example

```
{"op": "branch", "name": "main"}
{"op": "highlight", "commit": "c1"}
```

(`c1` is referenced but no commit with that id exists)

## How to fix

Declare the commit with a `commit` op before referencing it, correct
the typo, or — if you intended a branch — switch to the branch-name
field.
