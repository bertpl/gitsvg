# E200 - Reference to undefined branch

## Cause

A field that takes a branch name (`commit.branch`, `branch.from_branch`,
`merge.from`, `merge.into`, `remove.branches`) refers to a branch that has
not been declared at this point in the file.

When the same name exists as a commit id, the message hints that you
probably meant the commit-id field instead (e.g. `from_commit` rather
than `from_branch`).

## Example

```
{"op": "branch", "name": "main"}
{"op": "commit", "branch": "feat", "msg": "x"}
```

(`feat` is referenced but never declared)

## How to fix

Declare the branch with a `branch` op before referencing it, or correct
the typo. If the value is actually a commit id, switch to the commit-id
field (`from_commit` instead of `from_branch`, etc.).
