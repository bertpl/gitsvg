# E223 - table layout requires a vertical orientation

## Cause

The resolved theme has `commit_label_layout: table` together with a
horizontal `orientation` (`lr` or `rl`). Table layout places commit
metadata as fixed-width columns beside the graph, one row per commit —
a shape that only the vertical orientations (`bt`, `tb`) support. With
a horizontal orientation there is no well-defined "beside the graph"
column region, so the combination is rejected rather than rendered in a
way the author didn't intend.

## Example

```
{"op": "theme", "orientation": "lr", "commit_label_layout": "table"}
```

## How to fix

Pick one of:

- Use a vertical orientation: set `orientation` to `bt` or `tb` (or drop
  it to keep the default `bt`).
- Keep the horizontal orientation and use `commit_label_layout: inline`
  (the default free-floating labels).
