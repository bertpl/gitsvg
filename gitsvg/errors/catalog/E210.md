# E210 - pull_request `from` and `into` are the same branch

## Cause

A `pull_request` op specifies the same branch name for both `from`
and `into`. A PR conceptually proposes merging one branch into a
*different* branch — merging a branch into itself doesn't model any
real-world workflow.

## Example

```
{"op": "branch", "name": "main"}
{"op": "pull_request", "from": "main", "into": "main"}
```

## How to fix

Set `from` and `into` to two different branch names. If the intent
was to highlight or annotate work on a single branch, use
`highlight` (on a commit) instead.
