# E103 - Numeric value out of range

## Cause

A numeric field has a value outside the allowed range. Most numeric fields
in gitsvg are non-negative (slot counts, spacing, gaps), and negative
values are rejected.

## Example

```
{"op": "commit", "branch": "main", "msg": "x", "gap": -1}
```

(`gap` must be `>= 0`)

## How to fix

Use a value within the allowed range. The JSON Schema for the op
(`gitsvg schema <op>`) names the bound — `minimum`, `exclusiveMinimum`,
`maximum`, etc.
