# E202 - Branch name already declared

## Cause

A `branch` op tries to declare a name that's already in use by an
earlier `branch` op in the same effective input. Branch names are
unique within their namespace.

## Example

```
{"op": "branch", "name": "main"}
{"op": "branch", "name": "main"}
```

(`main` is declared twice)

## How to fix

Pick a unique name, or — if you intended to update the branch's
visuals — note that there is no in-place update operation on branches
in v0.0.x. Remove the original first if you want to redeclare it.
