# E106 - List too short

## Cause

A list field requires at least N items and the list is shorter (typically
empty). List fields like `replaces`, `commits`, and `branches` must
contain at least one entry when set — empty lists carry no meaning.

## Example

```
{"op": "remove", "commits": []}
```

(`remove.commits` must list at least one commit id)

## How to fix

Add the entries the list should hold. If the list would be empty, omit the
field entirely (when it's optional) rather than passing `[]`.
