# E217 - Empty theme op

## Cause

A `theme` op carries no `name`, no `keep_prior_overrides`, and no
explicit field override. With nothing to apply, the op is a no-op —
almost certainly a typo rather than the intended behaviour.

A `keep_prior_overrides` flag *does* count as content for the purpose
of this check (so it doesn't double-up with E220), but on its own it
will still be rejected — by E220 — for not having a `name` to modify.

## Example

```
{"op": "theme"}
```

## How to fix

Set `name` to select a built-in named theme, add one or more explicit
field overrides (e.g. `"background_color": "#fff"`), or delete the
empty op.
