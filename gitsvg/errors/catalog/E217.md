# E217 - Empty theme op

## Cause

A `theme` op carries neither a `name` nor any explicit field override.
With nothing to apply, the op is a no-op — almost certainly a typo
rather than the intended behaviour.

## Example

```
{"op": "theme"}
```

## How to fix

Set `name` to select a built-in named theme, add one or more explicit
field overrides (e.g. `"background_color": "#fff"`), or delete the
empty op.
