# E214 - remove of a branch blocked by an open pull_request referencing it

## Cause

A `remove` op tries to delete a branch that is still referenced by
one or more open pull-requests (as either `from` or `into`).
Removing the branch would orphan those PRs; the engine refuses
rather than silently cascading.

## Example

```
{"op": "branch", "name": "main"}
{"op": "branch", "name": "feat", "from_branch": "main"}
{"op": "pull_request", "id": "pr1", "from": "feat", "into": "main"}
{"op": "remove", "branches": ["feat"]}
```

(`pr1` still references `feat`)

## How to fix

Close the referencing pull-request(s) first:

```
{"op": "remove", "pull_requests": ["pr1"]}
{"op": "remove", "branches": ["feat"]}
```
