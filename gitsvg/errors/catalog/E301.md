# E301 - Import depth limit exceeded

## Cause

The chain of `import` ops nests deeper than the configured limit
(default 1000). The limit is a hard cap that bounds compilation time
even when no cycle exists — a chain 1001 files long is almost
certainly an accident.

## Example

A series of files each importing the next: A imports B, B imports C,
... reaching past 1000 levels of nesting.

## How to fix

Flatten the chain by inlining the deeper ops into a closer ancestor,
or — if the depth is genuinely needed — restructure into a tree of
shorter chains. If you have a legitimate use case for a higher limit,
that's a request for the maintainers.
