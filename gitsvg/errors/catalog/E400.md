# E400 - Branch root no longer exists

## Cause

A branch was declared with `from_branch` or `from_commit`, which
resolved to a specific commit at declaration time. That commit was
later removed (by a `remove` op or by another commit's `replaces:`
squash) and never restored. End-of-file validation flags the branch
as dangling.

## Example

```
{"op": "branch", "name": "main"}
{"op": "commit", "branch": "main", "id": "c1", "msg": "x"}
{"op": "branch", "name": "feat", "from_commit": "c1"}
{"op": "remove", "commits": ["c1"]}
```

(`feat` was rooted on `c1`; `c1` is gone at end-of-file)

## How to fix

Either restore the missing commit (re-add `c1` later in the file —
the rebuild pattern) or remove the branch that depends on it. If the
branch was supposed to follow a different root, redeclare it with the
right `from_branch` / `from_commit`.
