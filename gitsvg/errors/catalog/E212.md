# E212 - Duplicate open pull_request for the same (from, into) pair

## Cause

A `pull_request` op proposes a merge between two branches that
already have an open pull-request between them. Two simultaneous PRs
from the same source into the same target doesn't model a real-world
workflow (a forge would coalesce or reject the second).

## Example

```
{"op": "branch", "name": "main"}
{"op": "branch", "name": "feat", "from_branch": "main"}
{"op": "pull_request", "id": "pr1", "from": "feat", "into": "main"}
{"op": "pull_request", "id": "pr2", "from": "feat", "into": "main"}
```

(pr2 duplicates pr1's source/target pair)

## How to fix

Close the existing pull-request first via a `remove` op:

```
{"op": "remove", "pull_requests": ["pr1"]}
{"op": "pull_request", "id": "pr2", "from": "feat", "into": "main"}
```

Or, if both PRs were intended to coexist, point them at different
branches.
