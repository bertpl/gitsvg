# E104 - String pattern violation

## Cause

A string field's value does not match the required pattern. ID-like fields
(`name`, `id`, `branch`, `from_branch`, etc.) disallow whitespace and
empty strings via the pattern `^\S+$`. The `color` field on `branch`
requires a hex color like `#7b8fb2` or `#abc`.

## Example

```
{"op": "branch", "name": "feat with space"}
```

(`name` may not contain whitespace)

## How to fix

Use a value that matches the pattern. For ID-like fields, replace
whitespace with `-` or `_`. For colors, use a 3- or 6-digit hex code with
a leading `#`.
