# E211 - pull_request id already used

## Cause

A `pull_request` op specifies an explicit `id` that matches the id
of another pull-request still open in current state. PR ids must be
unique among open PRs.

## Example

```
{"op": "branch", "name": "main"}
{"op": "branch", "name": "feat", "from_branch": "main"}
{"op": "pull_request", "id": "pr1", "from": "feat", "into": "main"}
{"op": "pull_request", "id": "pr1", "from": "feat", "into": "main"}
```

(second `pr1` collides with the first)

## How to fix

Use a different `id`, or omit `id` to let the engine auto-generate
one. If the duplicate was unintentional, the earlier PR likely
should be closed via `remove` first.
