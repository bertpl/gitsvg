# E215 - Reference to undefined pull_request

## Cause

A `remove` op's `pull_requests:` list names an id that doesn't
match any open pull-request in current state. Either the id was
never declared, or the pull-request has already been removed.

## Example

```
{"op": "branch", "name": "main"}
{"op": "branch", "name": "feat", "from_branch": "main"}
{"op": "remove", "pull_requests": ["pr1"]}
```

(`pr1` was never declared)

## How to fix

Declare the pull-request first via a `pull_request` op, or remove
the dangling id from the `pull_requests:` list. If the typo was
the issue, correct the id.
