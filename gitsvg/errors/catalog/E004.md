# E004 - Line is not a JSON object

## Cause

The line parses as valid JSON, but the top-level value is not an object.
Each line in a `.gitsvg.jsonl` file must be a JSON object — lists, numbers,
strings, and booleans at the top level are rejected.

## Example

```
[{"op": "branch", "name": "main"}]
```

(the line is a JSON array, not an object — drop the surrounding brackets)

## How to fix

Replace the line with a single JSON object. If you have multiple operations,
put each on its own line.
