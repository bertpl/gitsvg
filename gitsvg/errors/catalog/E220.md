# E220 - Guide overshoot out of range

## Cause

The `theme` op set `guide_overshoot_in_rows` above `1`. The branch
guides only need to extend a small distance past the content edges
to look like they go *around* the topmost / bottommost commit dot
rather than stopping at it. Values past a single row push the guide
endpoints deep into the canvas margins (or beyond the canvas
entirely) without a visual justification.

## Example

```
{"op": "theme", "guide_overshoot_in_rows": 2.5}
```

## How to fix

Set `guide_overshoot_in_rows` to a value in the `[0, 1]` range. The
package default is `0.2` (one fifth of `commit_spacing`).
