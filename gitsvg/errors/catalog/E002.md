# E002 - Missing op field

## Cause

The line is a valid JSON object but has no `op` field. Every line in a
`.gitsvg.jsonl` file must declare which operation it represents via the
discriminator field `op`.

## Example

```
{"name": "main"}
```

(no `op` field — the parser cannot tell what operation this line is)

## How to fix

Add an `op` field with one of the recognised op names. Run
`gitsvg schema --list-ops` to see the full list.
