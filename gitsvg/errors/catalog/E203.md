# E203 - Commit id already used

## Cause

A `commit` or `merge` op specifies an explicit id (`id:` on `commit`,
`as:` on `merge`) that already exists in current state. Commit ids
are unique within their namespace.

When `replaces:` lists the same id, the new commit may reuse it
because the replaced commit is removed first — that case is allowed
and does not trigger this error.

## Example

```
{"op": "branch", "name": "main"}
{"op": "commit", "branch": "main", "id": "c1", "msg": "first"}
{"op": "commit", "branch": "main", "id": "c1", "msg": "second"}
```

(both commits ask for `c1`)

## How to fix

Use a unique id, or omit `id:` to auto-generate one. If you intend
to squash the previous commit, add `replaces: ["c1"]` to the new
commit — that vacates the id atomically.
