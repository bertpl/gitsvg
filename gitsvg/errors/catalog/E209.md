# E209 - merge `from` and `into` are the same branch

## Cause

A `merge` op specifies the same branch name for both `from` and
`into`. Merging a branch into itself models no real-world workflow:
both endpoints resolve to the same commit, so the "merge" would
collapse to an ordinary single-parent commit rather than joining two
lines of history. This mirrors the `pull_request` guard (E210).

## Example

```
{"op": "branch", "name": "main"}
{"op": "commit", "branch": "main", "id": "c1", "msg": "a"}
{"op": "merge", "from": "main", "into": "main"}
```

## How to fix

Set `from` and `into` to two different branch names — `merge` joins a
source branch into a distinct target. To add an ordinary commit on a
single branch, use `commit` instead.
