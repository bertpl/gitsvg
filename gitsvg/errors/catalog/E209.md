# E209 - replaces: new commit's parents reference a replaced commit

## Cause

`replaces:` rule 7 — the squashing commit's own `parents:` references
one of the commits being removed. The intent is contradictory: you
can't squash a commit and then keep an explicit pointer to it.

## Example

```
{"op": "branch", "name": "main"}
{"op": "commit", "branch": "main", "id": "c1", "msg": "first"}
{"op": "commit", "branch": "main", "id": "c2", "msg": "second"}
{"op": "commit", "branch": "main", "id": "c3", "msg": "squash", "replaces": ["c1", "c2"], "parents": ["c1"]}
```

(`c1` is in both `replaces` and `parents`)

## How to fix

Drop the replaced ids from `parents:`. If you actually need to keep
a link, you don't want a squash — use the decomposed form instead.
