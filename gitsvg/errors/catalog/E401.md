# E401 - Commit parent no longer exists

## Cause

A commit lists a parent in its `parents:` field that no longer exists
in current state at end-of-file. The parent was either explicitly
written and the target commit was later removed, or the parent was
auto-generated by a `merge` op (the from-branch's tip or the
into-branch's prior tip) and that tip commit has since been removed.

## Example

```
{"op": "branch", "name": "main"}
{"op": "commit", "branch": "main", "id": "c1", "msg": "x"}
{"op": "branch", "name": "feat", "from_branch": "main"}
{"op": "commit", "branch": "feat", "id": "f1", "msg": "x", "parents": ["c1"]}
{"op": "remove", "commits": ["c1"]}
```

(`f1` declared `c1` as a parent; `c1` is gone at end-of-file)

## How to fix

Restore the missing parent (re-add it later — the rebuild pattern),
remove the dependent commit, or rewrite the commit's `parents:` to
point at a surviving commit.
