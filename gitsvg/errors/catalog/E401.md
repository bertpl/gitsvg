# E401 - Commit parent no longer exists

## Cause

A commit's parent no longer exists in current state at end-of-file. A
commit's parents are structural — its chain parent (the prior commit on
its branch) plus the merged-in tip for a `merge` commit — so this means a
`remove` op deleted a commit that some surviving commit still descends
from: either its chain predecessor or a branch tip a `merge` consumed.

## Example

```
{"op": "branch", "name": "main"}
{"op": "commit", "branch": "main", "id": "c1", "msg": "x"}
{"op": "branch", "name": "feat", "from_branch": "main"}
{"op": "commit", "branch": "feat", "id": "f1", "msg": "x"}
{"op": "merge", "from": "feat", "into": "main", "as": "m1"}
{"op": "remove", "commits": ["f1"]}
```

(`m1` merged `feat`'s tip `f1`; `f1` is gone at end-of-file, so `m1` has a
parent that no longer exists)

## How to fix

Restore the missing commit (re-add it later — the rebuild pattern) or
remove the dependent commit.
