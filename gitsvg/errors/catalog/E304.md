# E304 - `import` is not the first op

## Cause

When an `import` op is present, it must be the first op in the file —
imports are preludes that build on a predecessor frame, so they sit at
the top conventionally and structurally.

## Example

```
{"op": "branch", "name": "main"}
{"op": "import", "path": "./prev.gitsvg.jsonl"}
```

(the import follows a `branch` op)

## How to fix

Move the `import` op to the very first non-blank line of the file.
Everything else stays in its original order.
