# E205 - replaces: commit not on the same branch

## Cause

`replaces:` rule 2 — every commit listed in `replaces:` must live on
the same branch as the new commit. A squash conceptually compacts a
range on one branch; spanning branches isn't a squash.

## Example

```
{"op": "branch", "name": "main"}
{"op": "branch", "name": "feat", "from_branch": "main"}
{"op": "commit", "branch": "main", "id": "m1", "msg": "first on main"}
{"op": "commit", "branch": "feat", "id": "f1", "msg": "first on feat"}
{"op": "commit", "branch": "main", "id": "m2", "msg": "squash", "replaces": ["m1", "f1"]}
```

(`f1` is on `feat`, not on `main` — the new commit's branch)

## How to fix

Use `replaces:` to compact a range on a single branch. For
cross-branch destructive edits, use the `remove` op directly.
