# E305 - Import path escapes the top-level file's directory

## Cause

An `import` op's `path` points outside the directory tree of the
top-level input file — by traversing upward with `..`, through a
symlink that resolves outside, or by being an absolute path (absolute
paths are always rejected). Imports are confined to the top-level
file's directory so that rendering an input file can only ever read
files from that file's own tree.

The check applies to the whole import chain: a nested import deep in a
chain must also stay inside the *top-level* file's directory.

## Example

```
{"op": "import", "path": "../outside.gitsvg.jsonl"}
```

(when the importing file is the top-level input, `../` leaves its
directory)

## How to fix

Place the imported file inside the top-level file's directory tree and
reference it with a relative path — a sibling (`./base.gitsvg.jsonl`)
or a subdirectory (`./shared/base.gitsvg.jsonl`).
