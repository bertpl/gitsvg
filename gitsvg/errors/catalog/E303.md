# E303 - Multiple `import` ops in one file

## Cause

A file declares more than one `import` op. The format only allows at
most one — files form chains (or trees of chains), never DAGs. The
chain pattern (one import = "build on top of the predecessor frame")
covers all known use cases.

## Example

```
{"op": "import", "path": "./a.gitsvg.jsonl"}
{"op": "import", "path": "./b.gitsvg.jsonl"}
{"op": "branch", "name": "main"}
```

## How to fix

Pick one of the imports — typically the immediate predecessor frame —
and drop the others. If you actually need to merge content from two
files, copy the relevant ops directly into the current file.
