# E100 - Unknown field

## Cause

The op contains a field name that the model does not recognise. Models are
strict (`extra="forbid"`), so unknown fields are rejected rather than
silently ignored — this catches typos and stale field names early.

## Example

```
{"op": "branch", "name": "main", "from_branche": "feat/x"}
```

(typo: `from_branche` instead of `from_branch`)

## How to fix

Check the field name against the op's schema. Run `gitsvg schema <op>` for
the full JSON Schema of any op, including the list of accepted fields.
