# E208 - replaces: external commit's parents reference a replaced commit

## Cause

`replaces:` rule 5 — a commit outside the replaced set has explicit
`parents:` referring to one of the commits you're trying to squash.
After the squash, the surviving commit's parents would dangle.

## Example

```
{"op": "branch", "name": "main"}
{"op": "commit", "branch": "main", "id": "c1", "msg": "first"}
{"op": "branch", "name": "feat", "from_branch": "main"}
{"op": "commit", "branch": "feat", "id": "f1", "msg": "feat", "parents": ["c1"]}
{"op": "commit", "branch": "main", "id": "c2", "msg": "squash", "replaces": ["c1"]}
```

(`f1` explicitly lists `c1` as a parent; squashing `c1` would orphan it)

## How to fix

Drop the explicit `parents:` reference if it's just expressing the
implicit lineage already encoded by branch position, or use the
decomposed form (`remove` + `commit`) and rebuild the dependent
commits with new parents.
