# E221 - branch_pos conflicts with auto_lane_change

## Cause

A `branch` op set an explicit `branch_pos` (pinning the branch to a
fixed lane) while the resolved theme has `auto_lane_change` enabled. The
two are mutually exclusive: a pin fixes a branch's lane for its whole
life, whereas `auto_lane_change` keeps every branch free to migrate
toward lower lanes as lower lanes free up. Honoring both at once is
contradictory, so the combination is rejected rather than silently
ignoring one of them.

The error is attributed to each pinned `branch` op's line; the
`auto_lane_change` value is resolved once per diagram (last write wins
across the `theme:` ops).

## Example

```
{"op": "theme", "auto_lane_change": true}
{"op": "branch", "name": "main"}
{"op": "branch", "name": "feature", "from_branch": "main", "branch_pos": 3}
```

## How to fix

Drop one side of the conflict: remove the `branch_pos` pin to let the
branch migrate automatically, or set `auto_lane_change` back to `false`
(its default) to keep manual lane pinning.
