# E107 - Inter-field constraint violated

## Cause

Two or more fields on the op are inconsistent. Examples:

- `branch` op has both `from_branch` and `from_commit` — only one of
  the two roots is allowed.
- `commit` op has neither `msg` nor `hash` — at least one is required.
- `remove` op has neither `commits` nor `branches`, or has both — exactly
  one is required.

## Example

```
{"op": "remove"}
```

(`remove` requires exactly one of `commits` or `branches`)

## How to fix

Reread the constraint in the error message and adjust the conflicting
fields accordingly. The model docstring (and the JSON Schema's `description`)
spells out which combinations are legal.
