# E204 - Branch root constraint violation

## Cause

Every branch except the first declared one must specify exactly one
of `from_branch` or `from_commit` to root the branch. The first
branch in a file (typically `main`) is the only one allowed to omit
both.

(Specifying both is a schema-level error caught earlier as E107.)

## Example

```
{"op": "branch", "name": "main"}
{"op": "branch", "name": "feat"}
```

(`feat` is not the first branch and specifies neither root field)

## How to fix

Add `from_branch` or `from_commit` to the offending branch op. Use
`from_branch` when you want the branch to fork off another branch's
tip, or `from_commit` when you have a specific commit id you want to
root on.
