# E302 - Imported file cannot be read

## Cause

The path on an `import` op cannot be opened — typically because the
file does not exist at the resolved location. Other failures (a
directory at that path, a permissions error, etc.) also surface here.

The path is resolved relative to the importing file, not the working
directory.

## Example

```
{"op": "import", "path": "./prev.gitsvg.jsonl"}
```

(when no `prev.gitsvg.jsonl` lives next to the importing file)

## How to fix

Check that the file exists at the path written and that the path is
relative to the importing file. The error message names the resolved
absolute path.
