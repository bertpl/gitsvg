# E213 - merge blocked by an open pull_request with the same (from, into) pair

## Cause

A `merge` op runs while an open pull-request still has the same
source/target pair. The locked PR lifecycle is explicit: open via
`pull_request`, close via `remove`, then `merge`. Allowing the
merge to silently consume the PR would obscure the lifecycle and
make diagrams harder to read.

## Example

```
{"op": "branch", "name": "main"}
{"op": "branch", "name": "feat", "from_branch": "main"}
{"op": "commit", "branch": "feat", "id": "f1"}
{"op": "pull_request", "id": "pr1", "from": "feat", "into": "main"}
{"op": "merge", "from": "feat", "into": "main"}
```

(`merge` runs while `pr1` is still open between the same branches)

## How to fix

Close the matching pull-request before the merge:

```
{"op": "remove", "pull_requests": ["pr1"]}
{"op": "merge", "from": "feat", "into": "main"}
```
