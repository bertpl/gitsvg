# E001 - Invalid JSON syntax

## Cause

The line could not be parsed as JSON. Common causes: missing quotes around
field names, trailing commas, single quotes instead of double quotes, or
unescaped characters inside a string.

## Example

```
{op: "branch", name: "main"}
```

(field names must be in double quotes)

## How to fix

Ensure each line is a valid JSON object. Field names and string values use
double quotes; lists and objects must close cleanly. The `python -m json.tool`
filter is a quick way to spot malformed lines locally.
