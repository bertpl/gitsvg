# E216 - Unknown theme name

## Cause

A `theme` op set `name` to a value that doesn't match any built-in
named theme. Only the names registered in the package's named-theme
table are accepted.

## Example

```
{"op": "theme", "name": "midnight"}
```

(no built-in theme called `"midnight"`; the package today only ships
`"default"`)

## How to fix

Drop the `name` field and use explicit field overrides instead, or set
`name` to one of the registered values. The accompanying error message
lists the currently-known names.
