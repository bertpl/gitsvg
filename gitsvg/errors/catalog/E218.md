# E218 - Theme spacing must be positive

## Cause

The `theme` op set `branch_spacing` or `commit_spacing` to zero. Both
spacings are strictly positive: every other position/size value with a
grid anchor (margins, the arc corner, label and pill offsets) is
derived as a ratio of one of them, so zero would collapse all
derived geometry to a single point and overlap every lane or row.

## Example

```
{"op": "theme", "branch_spacing": 0}
```

## How to fix

Set the spacing to a positive value. The package default is `100`
for `branch_spacing` and `50` for `commit_spacing`.
