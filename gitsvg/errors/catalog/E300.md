# E300 - Import cycle detected

## Cause

A chain of `import` ops loops back on itself, either directly (a file
imports itself) or transitively (A imports B, B imports A). Comparison
is on resolved absolute paths, so `./a.jsonl` and `../dir/a.jsonl`
referring to the same file are caught as the same node.

## Example

```
# A.gitsvg.jsonl
{"op": "import", "path": "./B.gitsvg.jsonl"}

# B.gitsvg.jsonl
{"op": "import", "path": "./A.gitsvg.jsonl"}
```

(running `gitsvg validate A.gitsvg.jsonl` reports the cycle `A -> B -> A`)

## How to fix

Restructure the imports so the chain terminates. One file is the
ancestor and the others build on top of it; circular dependencies have
no useful meaning in this format.
